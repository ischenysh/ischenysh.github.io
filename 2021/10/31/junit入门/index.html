junit单元测试

测试分类

  • 黑盒测试 : 不关注具体的代码实现, 只需要关注能否得到正确结果
  • 白盒测试 : 不只要关注能否得到正确结果, 还要关注具体的代码实现

junit使用

  • 步骤

    1. 定义一个测试类(测试用例)

      建议:

      • 测试类名 : 被测试的类 + Test
      • 包名 : 与被测试的类所在的包同级, 且取名为test

    2. 定义测试方法 : 可以独立运行

      建议:

      • 方法名 : test + 被测试的方法名
      • 返回值 : void
      • 参数列表 : 空参
    1. 给方法前加 @test 注解
    2. 导入junit 依赖环境

  • 判定结果

    • 红色 : 失败
    • 绿色 : 成功
    • 一般会使用断言操作

  • 补充

    • @Before :

      被注解的方法总是在测试方法之前执行

    • @After :

      被注解的方法总是在测试方法之后执行
